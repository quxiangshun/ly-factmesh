## 2. 企业版 MOM 功能全景（对标参考）

以下能力来自对 **Carbon Manufacturing System**、**MeMOM**、**Siemens Opcenter**、**Plex MES**、**ABB MOM** 及 **ISA-95 Level 3**、国内 MES 厂商实践的归纳，作为 LY-FactMesh 企业版能力目标参考。

### 2.1 总体架构（ISA-95 Level 3 视角）

| 层次 | 典型能力 | 说明 |
|------|----------|------|
| 生产运营管理 | 生产排程、工单执行、报工、在制品追踪 | MES 核心，计划→执行→反馈闭环 |
| 质量管理 | 检验计划、SPC、不合格处理、质量追溯 | QMS，与生产、物料、设备数据联动 |
| 库存与物流 | 入库、出库、拣配、齐套、批次/序列号 | WMS，支持多库位、多地点 |
| 设备与维护 | 设备状态、OEE、故障、预防性维护、CMMS 联动 | 设备物联 + 维护管理 |
| 制造智能 | 看板、KPI、报表、OEE/产能/质量分析 | 跨域聚合与可视化 |
| 集成与扩展 | ERP/PLM 对接、开放 API、多租户、审计 | 企业级集成与合规 |

## 3. 新版开发阶段总览

在保留现有 **P0–P4** 成果的前提下，将企业版能力按**阶段**与**优先级**拆解为 **E1–E5**，与 P 系列衔接，便于排期与验收。

| 阶段 | 名称 | 核心目标 | 建议周期 | 与现有计划关系 |
|------|------|----------|----------|----------------|
| E1 | 生产执行深化 | 工艺路线、工序、报工、在制品、排程雏形 | 10–14 周 | 在 P1 工单基础上扩展 |
| E2 | 仓储与质量深化 | 库位、入库/出库/盘点、检验计划、不合格、追溯 | 10–14 周 | 在 P1 WMS/QMS 基础上扩展 |
| E3 | 设备与 OEE | 告警、OEE、采集历史、维护对接（CMMS 可选） | 6–8 周 | 在 P1 IoT 基础上扩展 |
| E4 | 制造智能与报表 | 多维度看板、报表、导出、大屏/移动 | 6–8 周 | 在 P3 看板基础上扩展 |
| E5 | 企业集成与合规 | 多租户落地、操作日志、审计、ERP 对接、开放事件 | 6–10 周 | 与 P2/P3 安全与扩展结合 |

## 4. E1：生产执行深化

### 4.1 目标

- 工艺路线与工序模型落地，工单与工序关联。
- 报工（数量/工时/物料/质量）与在制品（WIP）追踪。
- 排程雏形：工单与设备/产线的时间窗展示与简单排产（可为人工排产+规则，不强制 APS 算法）。

### 4.2 里程碑与交付物

| 序号 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| E1-1 | 工艺与 BOM | 工艺路线、工序、BOM（可选版本）、与工单关联 | 工单可挂工艺路线，按工序执行 |
| E1-2 | 工序与派工 | 工序状态、派工记录、SOP 引用（可先 URL/文档） | 工序可派工、状态可更新 |
| E1-3 | 报工 | 报工单、报工记录（数量、工时、物料消耗、自检结果） | 支持按工序报工，数据入表 |
| E1-4 | 在制品 | WIP 实体、工序间移动、齐套检查（可选） | 在制品可查询、可移动 |
| E1-5 | 排程视图 | 工单–设备/产线–时间窗列表或甘特雏形 | 支持按日/班次查看排产，可调整顺序 |

### 4.3 任务清单（建议顺序）

1. **mom-mes**：工艺路线、工序、BOM 领域模型与表结构；工单关联工艺路线与工序。
2. **mom-mes**：工序派工、工序状态机、报工单与报工记录 API。
3. **mom-mes**：在制品（WIP）实体、工序间移动、与报工联动。
4. **mom-mes**：排程相关实体（产线/工作中心、时间窗）、排产列表/简单甘特 API。
5. **前端**：工序列表、报工录入、在制品查询、排产视图页。

### 4.4 输出

- 工单→工艺→工序→报工→在制品 闭环可演示。
- 排产视图可查看与简单调整，为后续 APS 预留接口。

## 5. E2：仓储与质量深化

### 5.1 目标

- WMS：仓库/库区/库位、入库单、出库单、盘点单、批次/序列号、库存预警。
- QMS：检验计划与检验项、检验结果、不合格处理流程、质量追溯链。

### 5.2 里程碑与交付物

| 序号 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| E2-1 | 仓库与库位 | 仓库、库区、库位、容量与状态 | 物料可绑定库位，库存按库位可查 |
| E2-2 | 入库与出库 | 入库单（采购/生产/退货）、上架；出库单、拣配、波次（可选） | 入库/出库流程可走通 |
| E2-3 | 盘点与批次 | 盘点单、盘点差异、批次/序列号主数据与库存关联 | 支持批次库存、先进先出策略 |
| E2-4 | 库存预警 | 安全库存、低/高/呆滞预警规则与通知 | 预警可配置、可查询与推送 |
| E2-5 | 检验计划与执行 | 检验计划、检验项、检验结果、合格/不合格判定 | 质检任务可挂检验计划，结果可录入 |
| E2-6 | 不合格与追溯 | 不合格单、处理（返工/让步/报废）、追溯链查询 API | 不合格可闭环；支持按批次/工单追溯 |

### 5.3 任务清单（建议顺序）

1. **mom-wms**：仓库、库区、库位实体与 API；库存表增加库位、批次。
2. **mom-wms**：入库单、上架逻辑；出库单、拣配与领料关联。
3. **mom-wms**：盘点单、差异处理；批次/序列号管理与 FIFO。
4. **mom-wms**：安全库存与预警规则、预警查询/通知。
5. **mom-qms**：检验计划、检验项、与质检任务关联；检验结果录入与判定。
6. **mom-qms**：不合格单、处理流程；追溯链（批次–工单–工序–物料–设备）建模与查询。
7. **前端**：库位管理、入库/出库/盘点页、检验执行页、不合格处理页、追溯查询页。

### 5.4 输出

- 入库→上架→出库→拣配→盘点 流程可演示；批次与预警可用。
- 检验计划→检验执行→不合格处理→追溯 闭环可演示。

## 6. E3：设备与 OEE

### 6.1 目标

- 设备告警规则与告警历史、通知（站内/邮件/企微可选）。
- 采集数据历史存储与查询；OEE 计算（可用率、表现率、良品率）及看板。
- 可选：与 CMMS 或简易维护模块对接（保养计划、工单）。

### 6.2 里程碑与交付物

| 序号 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| E3-1 | 告警 | 告警规则（阈值/表达式）、告警记录、通知渠道 | 超阈值可产生告警并通知 |
| E3-2 | 采集历史 | 采集项配置、时序存储与查询 API | 支持按设备/时间查询历史 |
| E3-3 | OEE | OEE 计算逻辑、按设备/班次/日汇总、OEE 看板 API | OEE 看板可展示 |
| E3-4 | 维护对接（可选） | 保养计划、维护工单或 CMMS 对接接口 | 可计划保养或对接外部 CMMS |

### 6.3 任务清单（建议顺序）

1. **mom-iot**：告警规则实体、规则引擎（阈值/简单表达式）、告警记录表、通知服务（邮件/站内）。
2. **mom-iot**：采集项配置、时序存储（可先用表或时序库）、历史查询 API。
3. **mom-iot**：OEE 计算（停机、产量、良品数来源与公式）、按设备/班次/日聚合 API。
4. **前端**：告警列表与规则配置、采集历史曲线、OEE 看板页。
5. **可选**：简易保养计划与工单或 Feign 对接 CMMS。

### 6.4 输出

- 告警可配置、可记录、可通知。
- OEE 看板与历史采集可演示；为预测性维护预留数据基础。

## 7. E4：制造智能与报表

### 7.1 目标

- 生产/设备/质量多维度看板（产线、工序、班次、日/周）。
- 报表：生产日报、质量报表、设备报表、可导出（Excel/PDF）。
- 大屏与移动端优化（布局、关键指标、移动报工/巡检入口）。

### 7.2 里程碑与交付物

| 序号 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| E4-1 | 多维看板 | 生产进度、设备 OEE/状态、质量合格率/不良 TOP | 看板数据来自各域聚合 API |
| E4-2 | 报表服务 | 生产日报、质量月报、设备报表等 API；导出服务 | 报表可生成并导出 |
| E4-3 | 大屏与移动 | 大屏布局与自动刷新；移动端报工/巡检入口 | 大屏可投屏；移动可报工/巡检 |

### 7.3 任务清单（建议顺序）

1. **各域**：提供统计与聚合 API（按产线、工序、班次、时间范围）。
2. **报表服务（可独立或网关侧聚合）**：报表定义、数据聚合、导出（Excel/PDF）。
3. **前端**：看板增强（ECharts/图表）、报表页与导出、大屏页、移动端适配与入口。

### 7.4 输出

- 多维度看板与报表可演示、可导出。
- 大屏与移动端具备基础可用性。

## 8. E5：企业集成与合规

### 8.1 目标

- 多租户：租户隔离落地、配额与开关。
- 操作日志与审计：关键操作落库、可查可追溯。
- ERP 对接：订单/BOM/工艺同步、完工回传（适配器模式）。
- 开放：事件总线或 webhook、订阅与推送。

### 8.2 里程碑与交付物

| 序号 | 里程碑 | 交付物 | 验收标准 |
|------|--------|--------|----------|
| E5-1 | 多租户落地 | 租户表、上下文注入、数据隔离、配额配置 | 多租户数据隔离可验证 |
| E5-2 | 操作日志与审计 | 操作日志表、关键接口埋点、审计查询 API | 关键操作可查 |
| E5-3 | ERP 对接 | 订单下发、BOM/工艺同步、完工回传接口与文档 | 至少 1 个 ERP 适配示例 |
| E5-4 | 事件与开放 | 领域事件发布、webhook 或消息推送、订阅管理 | 外部可订阅关键事件 |

### 8.3 任务清单（建议顺序）

1. **mom-admin**：租户实体、租户上下文（网关/Filter 注入）、各业务表 tenant_id 使用与索引。
2. **mom-admin**：操作日志表、AOP 或 Filter 埋点、审计查询 API。
3. **mom-common / 新模块**：ERP 适配器接口与示例（订单、BOM、完工）。
4. **网关或公共层**：领域事件发布、webhook 配置与推送、订阅管理 API。
5. **文档**：多租户设计、审计与合规说明、ERP 对接指南、事件订阅指南。

### 8.4 输出

- 多租户与审计可验收；ERP 对接与事件开放有示例与文档。

## 9. 与现有 P0–P4 的衔接

| 现有阶段 | 企业版阶段 | 衔接说明 |
|----------|------------|----------|
| P0 基础平台 | 不变 | 网关、Nacos、文档、前端骨架已就绪 |
| P1 核心业务 | E1/E2/E3 基础 | 工单、设备、物料、领料、质检任务已存在；E1/E2/E3 在其上深化 |
| P2 集成与优化 | E5 部分 | JWT、限流、监控已存在；E5 补多租户、审计、ERP、事件 |
| P3 生态与扩展 | E4 部分 | 看板与扩展文档已存在；E4 补报表、大屏、移动 |
| P4 持续运营 | 不变 | 版本发布、社区、问题修复持续进行 |

**建议实施顺序**：在保持 P4 持续运营的前提下，优先 **E1（生产执行深化）** → **E2（仓储与质量深化）**，再 **E3（设备 OEE）** 与 **E4（制造智能）**，最后 **E5（企业集成与合规）**。E1 与 E2 可部分并行（如 E2 的库位与入库可与 E1 的报工并行）。

## 10. 风险与依赖

| 风险 | 应对 |
|------|------|
| 工艺/BOM/排程模型复杂 | 先做最小可行模型（工艺路线+工序+工单关联），排程先做“视图+人工调整”，再迭代算法 |
| 追溯与批次跨多域 | 统一批次/序列号主键与关联规范，各域通过事件或 API 写入追溯链 |
| 多租户改造工作量大 | 新增表一律带 tenant_id；老表分批迁移，先读再写 |
| ERP 对接差异大 | 以适配器+文档为主，提供 1–2 个典型 ERP 示例，其余由实施侧扩展 |
